1) Variables

Affect(variable,valeur) :

Paramètres : variable  variable, valeur tout type
Fonction : associée une valeur à une variable.

Exemple :
Affect("client","sté VDOM")

Chaine(variable) :

Paramètres : variable  variable ou ("chaine","numeric","date","boolean")
Fonction : Convertir la valeur de la variable vers une chaine de caractères.

Exemple :
Chaine(unNombre)

ConvertirVers(variable,type) :

Paramètres : variable  chaine, type ("chaine","numeric","date","boolean")
Fonction : Permet si possible de convertir le type de départ de la variable vers un autre type, en général une chaine de caractères vers un type spécifique. Utile notamment pour utiliser les données transmises par un formulaire.

Exemple :
Affect("dateExpiration",ConvertirVers(Formulaire("expiration"),"date"))

Dictionaire(nom,clef,valeur,champliste,option) :

Paramètres : nom  chaine, clef chaine, valeur chaine,numeric,date,boolean, champliste (oui,non), option ("clef","valeur")
Fonction : Cette fonction permet de stocker et de récupérer des valeurs dans un dictionnaire global. Elle permet aussi de coupler le dictionnaire avec un champliste pour une alimentation automatisée par exemple. Le comportement en fonction des paramètres, est le suivant :

  • Si la chaine passée au paramètre valeur est vide: "", la clef est supprimée du dictionnaire.
  • Si le paramètre champliste est à oui la liste déroulante retournée par la fonction ChampListe(...) ayant comme nom le même que celui du dictionnaire sera automatiquement alimenté par celui-ci.
  • Cette alimentation peut-être faite soit pas la clef soit par la valeur, en fonction du paramètre option.

REMARQUE :
Lorsque le champ liste est alimenté par le dictionnaire, celle-ci ne peut plus être éditée, si une asymétrie apparaît entre le dictionnaire et la liste, celle-ci est automatiquement réinitialisée avec les données du dictionnaire.

Un dictionnaire dont toutes les clefs sont supprimées, est lui-même supprimé automatiquement.

Exemple :
Dictionnaire("societe","342 547 654","ma companie")

masociete = Dictionnaire("societe","342 547 654")

Dim(variable) :

Paramètres : variable  variable
Fonction : déclare une variable.

Exemple :
Dim("client")

2) Heures & Dates

Now() :

Paramètre : Aucun
Fonction : Retourne la date courante et l'heure

Date() :

Retourne la date courante sans l'heure

Jour(date) :

Paramètre : date  Date
Fonction : Retourne le jour de la date passée en paramètre.

Mois(date) :

Paramètre : date  Date
Fonction : Retourne le mois de la date passée en paramètre.

Annee(date) :

Paramètre : date  Date
Fonction : Retourne l'année de la date passée en paramètre.

Todate(yyyy,mm,dd) :

Paramètres : yyyy  numérique, mm  numérique, dd  numérique 
Fonction : convertit les trois nombres en une valeur de type date, ou yyyy est l'année, mm le mois et dd le jour.

3) Conditions

If(condition,si_vrai,si_faux) :

Paramètres : condition exp booléenne, si_vrai tout type, si_faux  tout type
Fonction : Cette fonction retourne une valeur en fonction de l'état de la condition qui se doit de retourner une valeur booléen true ou false. Si la valeur retournée est "goto ETIQUETTE" l'interpréteur de commande se branchera à l'étiquette définie, celle-ci doit se présenter dans la séquence de commande de la façon suivante :ETIQUETTE. 

Exemple : 
if(opFile="ADD","goto ADD","")

:ADD
   affect("corpsTXT","Vous avez reçu un nouveau Fax")

4) Opérations sur les fichiers

AnnulerSuppression() :

Paramètre : Aucun
Fonction : S'utilise en conjonction avec l'évènement DELETE après une commande Supprimer(). Cette fonction permet d'annuler la suppression du document.

CheminFichier() :

Paramètre : Aucun
Fonction : Retourne le chemin du fichier courant.

DeplacerVers(chemin) :

Paramètre : chemin chaine
Fonction : Déplace le fichier sélectionné vers l'emplacement indiqué par chemin. Si celui-ci n'existe pas, il est alors créé.

Exemple : 
DeplacerVers("/Archive/Client/2004")

Fichier_nom() :

Paramètre : Aucun
Fonction : Retourne le nom du fichier.

Fichier_lien() :

Paramètre : Aucun
Fonction : Retourne une chaine représentant le code HTML d'un lien vers le fichier

Image(chemin_image,hauteur,largeur) :

Paramètre : chemin_image chaine, hauteur  optionnel numérique, largeur  optionnel numérique
Fonction : Cette fonction retourne une chaine HTML représentant l'affichage d'une image. Le paramètre chemin image doit être le chemin absolu (depuis la racine) de l'image dans ProShare, les 2 autres paramètres permettent de contraindre la dimension de l'image. 

Exemple : 
affect("imageLogo",image("/resources/logo-transparent.png",60,60))

LienVersFichier(chemin_fichier,label_lien) :

Paramètre : chemin_fichier chaine, label_lien  chaine
Fonction : Cette fonction retourne une chaine HTML représentant un lien public (de type SmartFolder) vers un fichier stocké dans ProShare.  Il permet notamment de créer des mails contenant un lien vers un fichier téléchargeable.

Exemple : 
affect("LienPublic",LienVersFichier("/resources/catalogue.pdf","notre catalogue"))

NomFichier(extension) :

Paramètre : extension oui,non
Fonction : Retourne le nom du fichier courant.

Renommer(nouveau_nom) :

Paramétre : nouveau_nom chaine
Fonction : Permet de renommer le fichier courant avec la chaine passée en paramètre.

Exemple : 

Renommer("Facture pour le client " + Formulaire("client") +" du "+date())

Supprimer() :

Paramètre : Aucun
Fonction : Supprime le fichier en cours de traitement

5) Utilisateurs & droits

GestionDesDroits(utilisateur,droits) :


Paramètres : utilisateur  chaine, droits ("lecture","lecture/modification")
Fonction : Cette fonction permet de supprimer les droits courants du fichier en cours de traitement et de les remplacer par des droits en lecture ou lecture/modification des utilisateurs passés en paramètre. La liste des utilisateurs passée en paramètre doit être sous la forme d'une liste d'identifiants internes (guid) séparés par ";". Pour obtenir cette liste, les fonctions UtilisateurEnCours(), UtilisateursDuRépertoire() et TrouverLesUtilisateurs() doivent être utilisées.

Exemple : 
GestionDesDroits(UtilisateurEnCours(),"lecture/modification")

ListeDesUtilisateurs(nom_groupe) :

Paramètre : nom_groupe optionnel chaine
Fonction : Retourne la liste des identifiants internes (guid) des utilisateurs enregistés dans ProAdmin.

  • Si aucun paramètre : retourne toute la liste des utilisateurs
  • Si un paramètre : retourne la liste des utilisateurs du groupe passé en paramètre.

TrouverUtilisateurs(loginList) :


Paramètre : loginList chaine
Fonction : Cette fonction convertit une liste de login en une liste d'identifiants internes (guid).

Exemple : 
GestionDesDroits(TrouverUtilisateurs("monlogin"),"lecture")

Utilisateur(guid,champ) :


Paramètres : guid  chaine, champ  ("prénom","nom de famille","nom","email","téléphone")
Fonction : Retourne des informations sur un utilisateur enregistré dans ProAdmin. L'identifiant de l'utilisateur est fourni dans le paramètre guid et l'information à retourner par champ, qui peut prendre les valeurs suivantes :

·         Prénom : Retourne la valeur qui correspond au prénom

·         Nom de famille : Retourne le nom de famille

·         Nom : Retourne le nom complet sous la forme prénom nom de famille

·         Email : Retourne l'email

·         Téléphone : Retourne le champ téléphone

Exemple : 
Utilisateur(Formulaire("commercial"),"nom")

UtilisateursDuRepertoire(chemin,type) :

Paramètres : chemin  chaine,  type optionnel ("tous","utilisateurs","groupes") 
Fonction : Retourne une liste d'identifiants (guid) représentant l'ensemble des utilisateurs ayant le droit de lecture sur le répertoire passé en paramètres. Le paramètre type permet de choisir ceux qui seront retournés :

·         Tous : La fusion des groupes et des utilisateurs.

·         Utilisateurs : Seuls les utilisateurs explicitement définis.

·         Groupes : La fusion des utilisateurs des groupes.

Exemple : 
SendMail(UtilisateursDuRepertoire("../to","tous"),"Vous avez une notification !",message)

 

UtilisateurEnCours() :

Paramètres : Aucun 
Fonction : Retourne l'identifiant de l'utilisateur connecté.

6) Fonctions spécifiques

Abs(value) :

Paramètre : value numéric
Fonction : Retourne la valeur absolue d'une valeur.

Exemple : 
Affect("montantAbsolue",Abs(montantAbsolue))

BoutonVisible(visible,guid) :

Paramètre : visible (oui,non), guid optionnel  chaine
Fonction : Cette fonction permet de rendre visible ou invisible un bouton ou pas.  Le premier paramètre est une valeur booléenne qui si, elle est à oui affiche le bouton et inversement. Le bouton affecté est celui de la règle courante. Pour agir sur un autre bouton, il faut utiliser le deuxième paramètre guid.

Exemple : 
BoutonVisible(WF_State="PROCESSING")

Chemin(guid) :

Paramètre : guid chaine
Fonction : Cette fonction permet de retourner un chemin de type /rep1/rep2, à partir de l'identifiant fourni en paramètre. Cet identifiant est généralement généré depuis une liste déroulante 'cf: ChampListe' grâce à la fonction Formulaire. 

Exemple : 
DeplaceVers(Chemin(Formulaire("destination")))

EstNumeric(valeur) :

Paramètre : valeur chaine/numeric
Fonction : Cette fonction retourne oui si la valeur passée en paramètre est de type numérique. 

Exemple : 
if(EstNumeric(Formulaire("montantHT")),ConvertieVers(Formulaire("montantHT"),"numeric"),"goto Error")

EcrireDansHistorique(message) :

Paramètre : message chaine
Fonction : Cette fonction permet d'écrire un message dans l'historique du fichier. 

Exemple : 
EcrireDansHistorique("Document validé par :" + Utilisateur(UtilisateurEnCours(),"Nom") )

 

NonVide(valeur) :

Paramètre : valeur chaine
Fonction : Cette fonction retourne oui si la valeur passée en paramètre n'est pas vide. 

Exemple : 
if(NonVide(Formulaire("NumFacture")),"goto process","goto Error")

Partager(valeur,separateur,item) :

Paramètre : valeur chaine, separateur chaine, item chaine
Fonction : Cette fonction permet de partager la chaine de caractère. 

Exemple :
Affect("client",Partager(NomFichier(non),"_",1))

SendMail(destinataire(s),objet,corps) :

Paramètres : destinataire(s)  chaine, objet  chaine, corps  chaine
Fonction : La fonction sendmail supporte dans le paramètre destinataire(s) des mails multiples séparés par un caractère du type ";". Elle supporte aussi une liste d'utilisateurs internes qui sont caractérisés par leur guid. Il peut être utile par exemple de la combiner avec la fonction UtilisateursDuRepertoire. Le paramètre corps supporte les expressions encadrées par {}, l'expression ainsi encadrée sera évaluée. Le nom de l'émetteur sera Workflow.

Exemple : 
sendmail("utilisateur@domaine.com","Commande en cours","Votre commande a été traitée par notre service client.")

Windows(titre,action,message) :

Paramètres : titre chaine, action ("reload,""timer"), message chaine
Fonction : Affiche une fenêtre POP-Up qui permet d'informer l'utilisateur du résultat de l'action qu'il a fait. Le paramètre type peut avoir deux valeurs "reload" ou "timer". Si vous choisissez reload, cela permet de recharger la fenêtre et si vous choisissez "timer", le message sera affiché pendant 3 secondes.

Exemple : 
Windows("Lancement du traitement","timer","Vous avez validé le dossier de vente")

7) Fonctions pour ProSuite

CreeEvtProPlanning (date,calendrier,description,emails_Invitations) :

Paramètres : date date, calendrier chaine, description chaine, emails_Invitations chaine
Fonction : Cette fonction permet de créer dans le calendrier ProPlanning donné en paramètre, un évènement sur la journée dont la date est définie par le paramètre date. La description de l'évènement est donnée via le paramètre description. Il est possible de donner une liste d'emails pour leur envoyer des invitations, les emails de la liste doivent être séparés par ";".

Exemple : 
CreeEvtProPlanning(now()+340,"Contrats","Vous devez vérifier le contrat XXXXX")

PourRecherche (type,parameters) :

Paramètres : type chaine, parameters liste
Fonction : Cette fonction enregistre des métadonnées relatives au document pour recherche ultérieure dans ProSearch. Cette fonction s'utilise avec le Plug-In Workflow fonction "Documents". La liste des paramètres fonctionne par couple, le premier défini le nom de la métadonnée et le second sa valeur.

Exemple : 
PourRecherche("Factures","Montant HT",Formulaire("montantht"),"Fournisseur",Formulaire("fournisseur"))

Tache (operation,parameters) :

Paramètres : operation ("new","update","delete"), parameters liste
Fonction : Cette fonction enregistre dans ProSearch des informations dans la base documentaire qui sont associées au plug-in Workflow fonction "Mes tâches". Cette fonction est polymorphe et dépend de l'opération passée en paramètre.

  • new : Crée une nouvelle tâche et retourne un identifiant.
  • update : Met à jour une ou plusieurs métadonnées.
  • delete : Supprime la tâche.

1- Tache ("new",WF_priority,WF_name,WF_user,WF_groups,WF_limit,WF_step,WF_step_max,WF_comments)

Paramètres : operation = "new"

  • WF_priority  numérique : 1 (Faible), 2 (Normal), 3 (Important),4 (Urgent).
  • WF_name  chaine : Nom du Workflow s'affichant dans les résultats en haut dans le titre.
  • WF_groups  chaine : Chaîne représentant le groupe ou la personne habilité au traitement de cette étape.
  • WF_step_name  chaine : Nom de l'étape en cours.
  • WF_limit  date : date limite d'exécution.
  • WF_step  numérique : Numéro de l'étape en cours.
  • WF_step_max  numérique : Nombre maximal d'étapes.
  • WF_comments  chaine : Commentaires à afficher.

Exemple : 
Affect("WF_Task_ID",Tache("new" ,priority,"Gestion des documents entrants","Tous","à traiter",ConvertirVers(Formulaire("expiration"),"date"),1,2,Formulaire("commentaires")))


2- Tache ("update",WF_Task_ID,parameters)

Paramètres : operation = "update",  WF_Task_ID chaine, parameters liste
Fonction : Permet de mettre à jour une tâche avec les paramètres à modifier. WF_Task_ID est l'identifiant généré lors de la création.

Exemple : 
Tache("update" ,WF_Task_ID,"wf_step_name","En cours","wf_comments","Traitement en cours par l'utilisateur : " + Utilisateur(UtilisateurEnCours(),"nom"),"wf_groups",Utilisateur(UtilisateurEnCours(),"nom"))

3- Tache ("delete",WF_Task_ID)

Paramètres : operation = "update",  WF_Task_ID chaine
Fonction : Permet de supprimer une tâche dont l'identifiant est WF_Task_ID.

Exemple : 
Tache("update" ,WF_Task_ID,"wf_step_name","En cours","wf_comments","Traitement en cours par l'utilisateur : " + Utilisateur(UtilisateurEnCours(),"nom"),"wf_groups",Utilisateur(UtilisateurEnCours(),"nom"))

 

8) Fonctions pour le formulaire

Le formulaire est ce que l'utilisateur voit lorsqu'il appuie sur Action. Les fonctions ci-dessous permettent de le générer. Les éléments apparaissent les uns sous les autres dans l'ordre de leur apparition.

8.1 Les fonctions pour la création d'un formulaire

ChampTexte(nom,label,valeurParDefaut,obligatorie) :


Paramètres : nom chaine, label chaine, valeurParDefaut chaine, obligatoire (oui,non)
Fonction : Créer un élément de type champ de formulaire permettant de saisir une valeur. Le paramètre nom est celui qui doit être utilisé dans la fonction Formulaire pour récupérer la valeur saisie par l'utilisateur. Le paramètre label contient le texte à afficher à gauche du champ de formulaire. Le paramètre valeurParDefaut permet de remplir le champ avec une valeur prédéfinie. Ce champ sera utilement alimenté par une variable dans le cas de la reprise d'une valeur préexistante. Le paramètre obligatoire informe le système s'il doit faire une vérification sur ce champ, c'est-à-dire s'il doit contenir une valeur.

REMARQUE : Si la valeur par défaut est de type date alors le Champ se transformera automatiquement en un DatePicker permettant de sélectionner une date via un calendrier.

Exemple : 

ChampTexte("montantHT","Montant Hors Taxe","",oui)
ChampTexte("expiration","Expire à la date :",Date(),oui) > défini un champ de permettant de sélectionner une date.

ChampListe(nom,label,editable,valeursParDefaut,obligatorie) :

Paramétres : nom chaine, label chaine, editable ("oui","non"), valeursParDefaut chaine, obligatoire (oui,non)
Fonction : Créer un élément de type liste déroulante. Le paramètre nom est celui qui doit être utilisé dans la fonction Formulaire pour récupérer la valeur associée. Le paramètre label contient le texte à afficher à gauche du champ de formulaire.
Le paramètre valeurParDefaut possède plusieurs possibilités :

  • Lorsqu'il s'agit d'une suite de valeurs séparées par ; c'est cette liste qui va être affichée triée par ordre alphabétique.
  • Pour afficher la liste des utilisateurs, il faut utiliser la séquence  "['utilisateurs']" qui affichera tous les utilisateurs. S'il on veut afficher les utilisateurs d'un groupe en particulier il faut utiliser la séquence : "['utilisateur','nomDuGroupe']". ATTENTION ! lors de la récupération de l'utilisateur sélectionné, il faut utiliser une fonction spéciale pour récupérer son nom ou email. De cette manière Utilisateur(Formulaire("nomChampListe"),"email") par exemple.
  • Pour afficher la liste des répertoires, il faut utiliser la séquence  "['répertoires','/Chemin']" où /Chemin est le chemin à partir duquel la fonction va commencer à construire la structure des répertoires. Le niveau de profondeur est fixé à 3.
  • Pour utiliser des données externes stockées pour ce champ, il faut utiliser la séquence "['db']" ou "['db','valeur']". Le deuxième paramètre permet de positionner la liste sur une valeur.

Exemple : 

ChampListe("societe","Société :",oui,societe_encours,oui)

Commentaires(nblignes) :

Paramétres : nblignes numeric
Fonction : Cette fonction permet de créer un champ spécial couramment utilisé, le champ de commentaires. Il y a juste un seul paramètre à fournir : le nombre de lignes. Le champ de formullaire associé est prédéfini : commentaires. 


Exemple :
Commentaires(4)

CreerListe(fichier,séparateur,colonne,dbname,entête,trie) :

Paramétres : fichier chaine, séparateur chaine, colonne numeric, dbname chaine, entête (oui,non), trie (oui,non)
Fonction : Cette fonction permet à partir d'un fichier CSV d'alimenter une source de données pour le champ liste passé en paramètre dbname. Le chemin vers le fichier CSV (chemin absolu) est passé dans le paramètre fichier, les caractéristiques du fichier CSV sont :

  • Séparateur : Le caractère de séparation des données par exemple ;
  • Colonne : L colonne qui doit être importée ?
  • Entête : Le fichier dispose-t-il d'un entête (afin de l'ignorer lors de l'importation) ?
  • Trie : Les données doivent-elles être triées à posteriori ?

Exemple :
CreerListe("/data/mesclients.csv",";",1,"societe",non,oui)

PDFViewer(dimension) :

Paramétres : dimention numeric
Fonction : Cette fonction permet de visualiser le document de type uniquement PDF en cours de traitement. Suivant le paramétrage de la fonction SetParams(), la dimension représente la hauteur en pixel (valeur par défaut) ou la largeur si le mode pdfAgauche est oui. 

Exemple :
PDFViewer(600)

SetParams(titre,largeur,hauteur,pdfAgauche) :

Paramètres : titre chaine, largeur numérique, hauteur numérique, pdfAgauhe optionnel (oui,non)
Fonction : Permet de changer les paramètres par défaut de la fenêtre, le titre et sa hauteur et largeur. Le paramètre pdfAgauche n'est pas obligatoire. S'il est à oui, la fenêtre affichera automatiquement le PDF (si la commande PDFViewer est utilisée) à gauche et les champs s'aligneront à droite du document. A noter que la valeur de la fonction PDFViewer, passe de la définition de la hauteur à la largeur.

Exemple :
SetParams("Votre Workflow est formidable",800,600,oui)

Texte(texte,alignement) :

Paramètres : texte chaine, alignement ("gauche","centre","droit")
Fonction : Créer un élément de type Texte, le paramètre Texte contient la chaine à afficher qui peut utiliser des TAG HTML. Le paramètre Alignement  doit contenir la valeur gauche, centre ou droit qui définira la position d'affichage. 

Exemple : 
Texte("Formulaire de Saisie","centre")

8.2 Les fonctions pour le traitement du formulaire

Formulaire(nomDuChamp) :

Paramètres : nomDuChamp chaine
Fonction : Cette fonction retourne une chaine représentant la valeur saisie par l'utilisateur dans le champ passé en paramètre. 

Exemple : 
Formulaire("societe")

RafraichirProShare(option) :

Paramètres :  option (oui,non)
Fonction : Si option est à non, Proshare ne sera pas mis à jour, ce qui permet de garder la sélection du fichier. Cette fonction s'utilise en conjonction avec la fonction Windows option reload.

Exemple : 
RafraichirProShare(non)
Windows("Rechargement","reload","Nous rechargeons la page....")

9) Timers & Evénements

Les timers sont des évènements périodiques qui permettent d'exécuter des commandes périodiquement. Il existe 2 types de timers :

  • TIMER 1M : S'exécute toutes les minutes.
  • TIMER 1H : S'exécute toutes les heures.
  • TIMER 1D : S'exécute tous les jours.

Les autres événements sont  :

  • ADD : Se déclenche sur l'ajout d'un fichier sur le répertoire considéré
  • UPDATE : Se déclenche lors de la mise à jour d'un fichier
  • DELETE : Se déclenche uniquement après la fonction Supprimer(). La suppression manuelle d'un fichier ne le déclenche pas !

9.1 Les variables

evtUserGUID Variable:

Fonction : Cette variable contient l'identifiant de l'utilisateur ayant déclenché cet évènement. Il peut être pratique de la stocker pour envoyer un email après un traitement.

Exemple : 
Affect("Initiateur",evtUserGUID)

filter Variable:

Fonction : Avant d'exécuter les commandes principales du timer, l'exécution d'un système de filtre permet d'optimiser l'usage des timers. Pour déterminer si le résultat du filtrage autorise ou pas l'exécution des commandes principales, le moteur utilise la variable interne filter. En fonction de sa valeur (oui,non), les commandes sont exécutées ou pas :

  • oui : Le moteur exécute les commandes principales.
  • non : Le moteur stoppe l'exécution des commandes pour cet évènement.

Exemple : 
Affect("filter",oui)

9.2 Les fonctions de filtrage

AutoriserExtension(liste_extensions) :

Paramètres : liste_extensions chaine liste
Fonction : Retourne oui si le fichier courant contient une des extensions passées en paramètre. Cette fonction permet simplement de filtrer l'évènement sur des fichiers d'un ensemble d'extensions en particulier.

Exemple :
Affect("filter",AutoriserExtension("PDF"))

InterdireExtension(liste_extensions) :

Paramètres : liste_extensions chaine liste
Fonction : Retourne oui si le fichier courant ne contient pas une des extensions passées en paramètre. Cette fonction permet simplement de filtrer l'évènement sur tous les fichiers sauf ceux spécifiés.

Exemple :
Affect("filter",InterdireExtension("TMP","CHK","BAK"))

10) Analyse du document

Les fonctions d'analyses permettent d'extraire automatiquement des informations liées à des types de documents.

Analyse(doctype,paramètres) :

Paramètres : doctype chaine, paramètres liste
Fonction : Cette fonction permet d'analyser les documents en fonction de leur type passé en paramètre dans doctype. Les autres paramètres permettent de donner des informations complémentaires. Les documents disponibles pour analyse sont :

  •   Factures : doctype="factures", paramètres : numéroTVAintracom chaine, TauxTVA  optionnel numeric
    Le numéroTVAintracom est celui de la société qui analyse les documents. Cela permet d'être en mesure d'exclure des résultats son propre numéro dans la recherche et ainsi d'augmenter la pertinence. Le TauxTVA est optionnel. Il permet d'ajouter un taux non pris en compte automatiquement par le système d'analyse. Les résultats fournis par l'analyse sont :
    • tva_intracommunautaire : Retourne le numéro de TVA intracommunautaire de la facture, s'il existe.
    • pays : Le pays de l'émetteur de la facture.
    • rcs : le RCS ou l'identifiant de registre du commerce pour la société.
    • taux_tva : Le taux de TVA de cette facture.
    • montantht : Le montant hors taxes de cette facture.
    • montantttc : Le montant TTC de cette facture.
    • date : La date de la facture.
    • numero : Le numéro de la facture.

Exemple : 
Analyse("facture","FR75430163688")

Apprendre(identifiant,champ,valeur,supprime) :

Paramètres : identifiant chaine, champ chaine, valeur chaine, supprime (oui,non)
Fonction : Cette fonction permet au système d'apprendre par l'exemple. Certains champs nécessitent un apprentissage pour être reconnus car il n'existe pas pour eux une structure permettant à l'avance de les reconnaitre. C'est par exemple le cas pour le numéro de facture. Il est lié à chaque fournisseur et il n'existe pas de règle pour trouver à priori cette valeur. La fonction est très simple à utiliser. Il suffit pour une société donnée, de définir un identifiant (pour les sociétés française le RCS) dans le paramètre du même nom, puis de spécifier le champ nécessitant un apprentissage, dans notre exemple numéro. Pour finir, saisir la valeur  manuellement par l'opérateur. Le système, à partir de cet exemple et des autres, va essayer de trouver une règle de reconnaissance de façon automatique. Dans le cas où une valeur contiendrait par exemple une erreur de saisie, le paramètre supprime affecté à oui, permet de le supprimer.

Les champs avec apprentissage possible, reliés à la fonction Analyse() sont :

  • doctype="factures"
    • numero : Apprentissage des numéros de factures.

Exemple : 
Apprendre(ResultatRecherche("analyse:facture","rcs"),"factures:numero",Formulaire("numfacture"))

 

ResultatRecherche(fonctionderecherche,champ) :

Paramètres : fonctionderecherche chaine, champ chaine
Fonction : Cette fonction permet de récupérer les résultats des fonctions de recherche Analyse() & Trouve(). Le premier paramètre permet de définir la fonction et le second le champ à lire. La liste des fonctions de recherche supportées avec leurs champs, est :

  • fonctionderecherche="analyse:facture"
    • tva_intracommunautaire : Numéro de TVA intracommunautaire de la facture.
    • taux_tva : Taux de TVA trouvé par le moteur de recherche lors de l'analyse.
    • pays : Pays détecté dans le numéro de TVA intracommunautaire.
    • rcs : Le numéro RCS de la société si celle-ci est Française.
    •  montantht : Le montant Hors Taxe de la facture.
    • montantttc : Le montant TTC de la facture basé sur le principe que l'écart entre les deux valeurs (HT & TTC) est compatible avec un taux de TVA disponible. Il est possible d'ajouter des montants de TVA non conventionnel lors de l'analyse.
    • date : La date la plus probable de la facture.
    • numero : Le numéro de la facture déduit sur la base d'un apprentissage par l'exemple en utilisant la fonction Apprendre().
  • fonctionderecherche="trouve"
    • rcs : Le numéro RCS de la société si celle-ci est Française.
    • regexp : le résultat de l'expression régulière.

Exemple : 
ChampTexte("numfacture","Numéro de Facture :",ResultatRecherche("analyse:facture","numero"),oui)

SupprimeApprentissage(identifiant) :

Paramètres : identifiant chaine
Fonction : Cette fonction supprime les données d'apprentissage pour un identifiant donné.

Exemple :

:DELETE_RECORD
SupprimerApprentissage(ResultatRecherche("analyse:facture","rcs"))
Windows("Suppression apprentissage","timer","Suppression des données pour d'apprentissage")
end

Trouve(champ) :

Paramètres : champ chaine
Fonction : Cette fonction permet de rechercher une valeur en particulier ou le résultat d'une expression régulière si le champ ne correspond à aucun mot clef connu.

Exemple :

if(if(Trouve("RCS"),Dictionnaire("societe",ResultatRecherche("trouve","rcs")),"")="","goto ASK_DATA","goto NEXT_STEP")
end
